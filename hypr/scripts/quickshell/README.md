# Quickshell Shell

A QtQuick/Quickshell-based desktop shell for Hyprland — replaces the legacy
`cyber-shell` python taskbar with a fully-animated QML TopBar plus a stack of
floating panels (network, music, calendar, volume, wallpaper picker, etc.).

Originally adapted from
[ilyamiro/nixos-configuration](https://github.com/ilyamiro/nixos-configuration),
trimmed and customised for this machine.

---

## What lives here

| Path                          | Purpose                                                       |
| ----------------------------- | ------------------------------------------------------------- |
| `Main.qml`                    | Master overlay window. Hosts all popup panels + notifications |
| `TopBar.qml`                  | The always-on top bar (workspaces, clock, system stats, tray) |
| `Lock.qml`                    | Lock-screen overlay                                           |
| `ScreenshotOverlay.qml`       | Screen-recording / screenshot overlay                         |
| `WindowRegistry.js`           | Maps panel names → QML file + size + position                 |
| `Scaler.qml`                  | Resolution-aware scaling helper                               |
| `MatugenColors.qml`           | Reads `qs_colors.json` (generated by Matugen from wallpaper)  |
| `colors.json.template`        | Matugen template that produces `qs_colors.json`               |
| `qs_colors.json`              | Live Material-You palette (regenerated on wallpaper change)   |
| `workspaces.sh`               | Watches Hyprland workspace events → `/tmp/qs_workspaces.json` |
| `watchers/`                   | One small bash watcher per data source (audio, bt, sys, …)    |
| `calendar/` `music/` `network/` `volume/` `wallpaper/` `notifications/` `settings/` `monitors/` `focustime/` `guide/` `updater/` `stewart/` | Individual panel sub-modules |

## TopBar layout (left → right)

1. **Help · Search · Settings · Update** — quick-action icons
2. **Workspaces** — live Hyprland workspaces
3. **Media player** — current MPRIS track + transport
4. **Center**: clock + date + weather
5. **System Stats Pill** *(custom — replaces python taskbar)*  
   `CPU% │ RAM% │ CPU°C │ GPU% │ Disk% │ Mic │ Cam │ VPN │ Timer │ UPS │ Tasks │ Notifs`
6. **System Tray** (StatusNotifierItem)
7. **Keyboard · WiFi/Eth · Bluetooth · Volume · Battery**
8. **Recording indicator** (visible only while recording)

### Stats pill interactions

| Module        | Left click                          | Right click       | Scroll               |
| ------------- | ----------------------------------- | ----------------- | -------------------- |
| GPU           | open Monitors panel                 | —                 | —                    |
| Mic           | toggle mute                         | open `pavucontrol` | ±5 % volume         |
| VPN           | mullvad connect / disconnect        | —                 | —                    |
| Timer         | start / pause                       | reset             | ±1 minute duration   |
| Tasks         | launch terminal with `btop`         | —                 | —                    |
| Notifications | open Notification Center            | —                 | —                    |

CPU / RAM / Temp / Disk / Camera / UPS show data on hover (tooltip) only.

## Data plumbing

The TopBar polls a tiny set of bash watchers — **never** scrapes anything
heavy in QML directly. Each module follows the same pattern:

```
Process: …/watchers/<thing>_fetch.sh   →  StdioCollector → updates property
Process: …/watchers/<thing>_wait.sh    →  blocks via inotifywait → re-fires fetch
```

Special cases:
- **`watchers/sys_stats.sh`** is consolidated: emits ONE JSON every 2 s
  with everything the legacy python taskbar exposed (CPU, RAM, temp, disk,
  GPU/VRAM/temp, mic vol/mute, camera, VPN, UPS, timer, notif-count).
- **`watchers/timer_ctl.sh`** is a state-machine for the Pomodoro timer.  
  State file: `~/.cache/qs_timer_state.json`.
- **`watchers/battery_*.sh`** are no-op stubs on desktop chassis (no `/sys/class/power_supply/BAT*`).

## Autostart

Defined in `~/.config/hypr/autostart.conf`:

```conf
exec-once = bash -c "sleep 3 && qs -p ~/.config/hypr/scripts/quickshell/Main.qml" &
exec-once = bash -c "sleep 3 && qs -p ~/.config/hypr/scripts/quickshell/TopBar.qml" &
```

`~/.config/hypr/scripts/qs_manager.sh` also acts as a watchdog — every
panel toggle silently respawns either process if it has died.

## Manual reload / debug

```bash
# Reload the bar only (after editing TopBar.qml)
pkill -f "quickshell.*TopBar\.qml"
qs -p ~/.config/hypr/scripts/quickshell/TopBar.qml &

# Reload everything
pkill -f "quickshell.*Main\.qml"
pkill -f "quickshell.*TopBar\.qml"
qs -p ~/.config/hypr/scripts/quickshell/Main.qml  &
qs -p ~/.config/hypr/scripts/quickshell/TopBar.qml &

# Lint a QML file
qmllint ~/.config/hypr/scripts/quickshell/TopBar.qml

# See live debug output
qs -p ~/.config/hypr/scripts/quickshell/TopBar.qml   # foreground
```

## Adding a new module to the Stats Pill

1. Add a field in `watchers/sys_stats.sh` and emit it in the final JSON.
2. Add a `property` in `TopBar.qml` (search for `// ===== System-stats fields`).
3. Read it in the poller's `JSON.parse` block (search for `sysStatsPoller`).
4. Add a `StatTile { … }` inside `statsLayout` (search for `// SYSTEM STATS PILL`).

Pattern for a new tile:

```qml
StatSep {}
StatTile {
    icon: "<nerd-font glyph>"
    value: barWindow.<your_property> + "%"
    iconColor: mocha.<color>
    valColor:  statsPill.usageColor(barWindow.<your_property>, false)
    tip: "<your tooltip>"
    onActivated: () => Quickshell.execDetached(["bash","-c","<click cmd>"])
}
```

`mocha` colours come from `MatugenColors.qml` and re-skin themselves on
every wallpaper change.
